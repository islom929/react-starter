# React Starter

Feature-based React starter template.

## Tech Stack

- React 18 + TypeScript + Vite 5
- React Router v6
- TanStack React Query v5 (server state)
- Zustand (client state)
- Axios (HTTP + interceptors)
- Tailwind CSS 4 + shadcn/ui (new-york style)
- React Hook Form + Zod (form validation)
- i18next (uz, ru, uz_cyr)

### Barrel export rules

- **Feature subfolders** (api/, hooks/, types/, schemas/, constants/) в†’ have `index.ts` for short imports
- **Feature root** в†’ NO `index.ts` (no feature-level barrel export)
- **Components** в†’ NO `index.ts` вЂ” import each component directly by file path

This means imports look like:
```tsx
// Subfolder barrel вЂ” short
import { useProducts } from '@/features/products/hooks'
import { PRODUCTS_KEY } from '@/features/products/constants'
import type { Product } from '@/features/products/types'

// Components вЂ” direct file import
import { ProductForm } from '@/features/products/components/product-form'
```

## Rules

### Component placement
- Used in 1 place в†’ colocate it (`pages/home/components/` or `features/products/components/`)
- Used in 2+ places в†’ lift to `components/`
- Layout-specific (header, footer) в†’ inside `layouts/[name]/`

### Feature rules
- Each feature is self-contained: api, components, constants, hooks, types, schemas
- Features must not import other features directly вЂ” use shared layer

### Page rules
- Pages are composition only вЂ” import and arrange features
- No business logic in pages вЂ” keep it in features

### shared/ rules
- Infrastructure code only: lib, hooks, types, locales, store
- Frequently imported items (components) do NOT belong here

### shadcn/ui
- `components/ui/` is generated by shadcn CLI вЂ” do not modify
- Add new components: `npx shadcn add [component]`
- Custom wrappers go in `components/form/`

## Dialog System

Context-based stack dialog system. Call `useDialog()` from any component:

```tsx
const { open, close, closeAll } = useDialog()

open(
  <DialogWrapper title="Confirm">
    <p>Content</p>
    <button onClick={close}>Close</button>
  </DialogWrapper>
)
```

- Stack-based: supports dialog-on-top-of-dialog
- Single overlay: background does not darken with each stacked dialog
- Pass TSX directly, `close` is available in the same scope
- ESC key closes the topmost dialog
- Click outside closes the topmost dialog
- `DialogWrapper` provides title, close button, optional icon

- **Server state** в†’ React Query (`useQuery`, `useMutation`)
- **UI state** в†’ Context (dialog)
- **Global app state** в†’ Zustand (language)
- Do not use Zustand when React Query is sufficient

## Adding a New Feature

1. Create `src/features/[name]/` directory
2. Add subdirectories: `api/`, `components/`, `constants/`, `hooks/`, `types/`, `schemas/`
3. Each subfolder gets single files (`[name].[type].ts`) and an `index.ts` barrel export
4. Components do NOT get `index.ts` вЂ” import directly by file path
5. Create page at `pages/[name]/`, import from feature subfolders
6. Add route in `app/router.tsx`

## Adding a shadcn Component

```bash

```

Component lands in `components/ui/`. Do not edit вЂ” write a wrapper instead.
